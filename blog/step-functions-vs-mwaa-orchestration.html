Step Functions vs MWAA vs EventBridge: Orchestration Cost Comparison

The MWAA environment that cost $1,200/month doing nothing

A team running 12 daily Glue ETL jobs migrated from self-managed Airflow to Amazon MWAA for the managed infrastructure. The migration went smoothly. Six months later, the platform team reviewed the bill: MWAA was $1,187/month. The actual Glue job compute was $340/month. They were spending 3.5× more on the orchestrator than the actual work. For 12 daily jobs that each run 10–20 minutes, MWAA is a significant overspend.

The three orchestration models

AWS Step Functions charges per state transition: $0.025/1,000 transitions for Standard Workflows, $0.00001/state transition for Express Workflows. Amazon MWAA charges per environment hour: a mw1.small is $0.49/hour ($356/month), mw1.medium is $0.98/hour ($712/month). EventBridge Pipes charges per event processed: $0.40/million events — essentially free for typical pipeline volumes under 10M events/month.

Step Functions: when the state machine model fits

Step Functions is the right choice when your pipeline has conditional logic, retry policies that differ per step, parallel fan-out with synchronization, or human approval gates. The Amazon States Language (ASL) maps directly to these patterns. A Glue job that branches based on the output of a data quality check — run vs reject — is a natural Step Functions use case.

⚠ Watch out

Step Functions Standard Workflow state transitions accumulate fast in Map states. A Map state processing 10,000 items with 5 states each = 50,000 transitions = $1.25 per execution. Run this hourly and it is $900/month just for the orchestrator. Use Express Workflows for high-volume Map operations — the pricing cap makes them cost-effective for short, frequent executions.

MWAA: the right choice for complex DAG dependency graphs

MWAA earns its cost when you have: 50+ DAGs with cross-DAG dependencies (TriggerDagRunOperator patterns), existing Airflow code and plugins from a self-managed environment, SLA monitoring with email callbacks, or data-aware scheduling with Dataset triggers. The managed service eliminates scheduler high availability, executor configuration, and Celery/Kubernetes worker management. At 50+ DAGs, the operational savings justify the cost.

# MWAA DAG: data-aware scheduling with Dataset triggers
from airflow import DAG, Dataset
from airflow.operators.trigger_dagrun import TriggerDagRunOperator
from airflow.providers.amazon.aws.operators.glue import GlueJobOperator
from datetime import datetime

bronze_dataset = Dataset("s3://datalake/bronze/events/")
silver_dataset = Dataset("s3://datalake/silver/events/")

with DAG(
    dag_id="bronze_ingestion",
    schedule="@hourly",
    start_date=datetime(2026, 1, 1),
) as bronze_dag:
    GlueJobOperator(
        task_id="ingest_to_bronze",
        job_name="bronze-events-ingestion",
        outlets=[bronze_dataset],  # signals this dataset is updated
    )

with DAG(
    dag_id="silver_transformation",
    schedule=[bronze_dataset],  # runs when bronze_dataset is updated
    start_date=datetime(2026, 1, 1),
) as silver_dag:
    GlueJobOperator(
        task_id="transform_to_silver",
        job_name="silver-events-transform",
        outlets=[silver_dataset],
    )

EventBridge Pipes: the overlooked tool

For simple linear pipelines — Kinesis stream → Lambda → SQS → Glue — EventBridge Pipes eliminates the orchestration layer entirely. There is no scheduler to maintain. The pipe triggers on source events, applies an optional filter and enrichment Lambda, and delivers to the target. At $0.40/million events, a pipeline processing 5M events/day costs $60/month in EventBridge costs — vs $356/month minimum for MWAA.

✓ Decision framework

Use EventBridge Pipes for event-triggered linear pipelines with no branching. Use Step Functions when you need conditional logic, retries, or parallel branches — and you have fewer than ~1M state transitions/month. Use MWAA when you have 50+ DAGs, existing Airflow code, or need cross-DAG dependency management that Step Functions cannot model cleanly.

Cost at 100 daily pipeline runs

For a team running 100 daily pipeline executions, each with 20 states and 5-minute average duration: Step Functions Standard costs $0.025 × (100 × 20 transitions) / 1,000 = $0.05/day = $1.50/month. MWAA mw1.small costs $356/month flat — the same regardless of whether you run 100 or 10,000 DAG runs. Step Functions wins decisively below 200 DAGs at normal concurrency.
